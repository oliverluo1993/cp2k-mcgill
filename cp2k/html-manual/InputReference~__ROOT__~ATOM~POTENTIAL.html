Keyword descriptions

CONFINEMENT_TYPE
CONFINEMENT_TYPE (NONE|POLYNOM|BARRIER)
Define functional form of confinement potential.
This keyword expects a keyword. Default value: POLYNOM
valid keywords:
  • NONE: Do not use confinement potential
  • POLYNOM: Use polynomial confinement potential: a*R^b
  • BARRIER: Use a smooth barrier potential: a*F[r-ron)/w]
CONFINEMENT
CONFINEMENT prefactor range exponent (POLYNOM)CONFINEMENT prefactor range r_onset (BARRIER)
Definition of parameters for confinement potential
This keyword expects a list of reals. Default value: 0.0000000000000000E+00 0.0000000000000000E+00 0.0000000000000000E+00
PSEUDO_TYPE
PSEUDO_TYPE (NONE|GTH|UPF|ECP)
Pseudopotential type
This keyword expects a keyword. Default value: NONE
valid keywords:
  • NONE: Do not use pseudopotentials
  • GTH: Use Goedecker-Teter-Hutter pseudopotentials
  • UPF: Use UPF norm-conserving pseudopotentials
  • ECP: Use ECP semi-local pseudopotentials
POTENTIAL_FILE_NAME
POTENTIAL_FILE_NAME
Name of the pseudo potential file, may include a path
This keyword expects a string. Default value: POTENTIAL
