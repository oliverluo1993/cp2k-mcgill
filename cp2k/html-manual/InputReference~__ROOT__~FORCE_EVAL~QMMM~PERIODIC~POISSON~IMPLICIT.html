Keyword descriptions

BOUNDARY_CONDITIONS
BOUNDARY_CONDITIONS
Specifies the type of boundary conditions. Dirichlet=fixed value, Neumann=zero normal deriv. Mixed and Neumann boundaries essentially requires FFTW3 so that all grid sizes are FFT-able.
This keyword expects a keyword. Default value: PERIODIC
valid keywords:
  • PERIODIC: periodic boundary conditions
  • MIXED: Dirichlet + homogeneous Neumann boundary conditions
  • MIXED_PERIODIC: Dirichlet + periodic boundary conditions
  • NEUMANN: homogeneous Neumann BC (zero-average solution)
ZERO_INITIAL_GUESS
ZERO_INITIAL_GUESS
Whether or not to use zero potential as initial guess.
This keyword expects a logical
This keyword behaves as a switch T . Default value: F

max_iter

TOL
tol
Stopping tolerance.

OR_PARAMETER
OR_PARAMETER
Over-relaxation parameter (large epsilon requires smaller omega ~0.1).
This keyword expects a real. Default value: 1.0000000000000000E+00
variants: OMEGA
NEUMANN_DIRECTIONS
NEUMANN_DIRECTIONS
Directions in which homogeneous Neumann conditions are imposed. In the remaining directions periodic conditions will be enforced. Having specified MIXED or NEUMANN as BOUNDARY_CONDITIONS, the keyword is meant to be used to combine periodic and homogeneous Neumann conditions at the boundaries of the simulation cell.
This keyword expects a keyword. Default value: XYZ
valid keywords: XYZ, XY, XZ, YZ, X, Y, Z
