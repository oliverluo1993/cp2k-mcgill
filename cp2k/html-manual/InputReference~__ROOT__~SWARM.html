Section SWARM

Index of all sections. This section is located at __ROOT__%SWARM.

Section to control swarm runs. The swarm framework provides a common ground for master/worker algorithms. This section can not be repeated.

Keyword descriptions

BEHAVIOR
BEHAVIOR
Which behaviour should control the swarm.
This keyword expects a keyword. Default value: GLOBAL_OPT
valid keywords:
  • GLOBAL_OPT: Runs global geometry optimisation
NUMBER_OF_WORKERS
Number of workers used for swarm. Of the total number of processors one is used for the master, the remaining processors should be divisible by the number of workers.
This keyword expects an integer
REPLAY_COMMUNICATION_LOG
REPLAY_COMMUNICATION_LOG
Filename of communication log of previous run. Use this to restart a swarm.
This keyword expects a string. Default value: swarm_translog_replay.xyz
MAX_ITER
The maximum number iterations the master should perform
This keyword expects an integer. Default value: 2147483647
