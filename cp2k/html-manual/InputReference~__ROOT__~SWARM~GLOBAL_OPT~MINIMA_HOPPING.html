Section MINIMA_HOPPING

Index of all sections. This section is located at __ROOT__%SWARM%GLOBAL_OPT%MINIMA_HOPPING.

Section controlling the Minima Hopping method. This section can not be repeated.

This section cites following reference: [Goedecker2004]

Keyword descriptions

BETA_1
Factor used to increase temperature when escape failed, should be larger than 1.
This keyword expects a real. Default value: 1.1000000000000001E+00
BETA_2
Factor used to increase temperature when escape found known minima, should be larger than 1.
This keyword expects a real. Default value: 1.1000000000000001E+00
BETA_3
Factor used to decrease temperature when escape succeeded, should be smaller than 1.
This keyword expects a real. Default value: 9.0909090909090906E-01
ALPHA_1
Factor used to decrease acceptance energy, when minima was accepted, should be smaller than 1.
This keyword expects a real. Default value: 9.7999999999999998E-01
ALPHA_2
Factor used to increase acceptance energy, when minima was rejected, should be larger than 1.
This keyword expects a real. Default value: 1.0204081632653061E+00
E_ACCEPT_INIT
Initial value of acceptance energy
This keyword expects a real. Default unit: [hartree], default value: 5.0000000000000001E-03 hartree
TEMPERATURE_INIT
Initially temperature in Kelvin
This keyword expects a real. Default value: 1.0000000000000000E+02
SHARE_HISTORY
If set all worker will use a single share history of visited minima.
This keyword expects a logical
This keyword behaves as a switch T . Default value: F
