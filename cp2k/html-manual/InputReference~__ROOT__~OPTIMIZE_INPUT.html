Section OPTIMIZE_INPUT

Index of all sections. This section is located at __ROOT__%OPTIMIZE_INPUT.

describes an input optimization job, in which parameters in input files get optimized. This section can not be repeated.

Keyword descriptions

METHOD
METHOD FORCE_MATCHING
What kind of input optimization to perform.
This keyword expects a keyword. Default value: FORCE_MATCHING
valid keywords:
  • FORCE_MATCHING: Perform a force matching minimization.
ACCURACY
ACCURACY 0.00001
Final accuracy requested in optimization (RHOEND)
This keyword expects a real. Default value: 1.0000000000000001E-05
STEP_SIZE
STEP_SIZE 0.005
Initial step size for search algorithm (RHOBEG)
This keyword expects a real. Default value: 5.0000000000000003E-02
MAX_FUN
MAX_FUN 1000
Maximum number of function evaluations
This keyword expects an integer. Default value: 5000
ITER_START_VAL
ITER_START_VAL 0
Used for restarting, starting value of the iteration
This keyword expects an integer. Default value: 0
RANDOMIZE_VARIABLES
RANDOMIZE_VARIABLES 20
Percentage randomization of the free variables applied initially
This keyword expects a real. Default value: 0.0000000000000000E+00
