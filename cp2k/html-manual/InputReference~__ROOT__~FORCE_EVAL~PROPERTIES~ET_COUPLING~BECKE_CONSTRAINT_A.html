Section BECKE_CONSTRAINT_A

Index of all sections. This section is located at __ROOT__%FORCE_EVAL%PROPERTIES%ET_COUPLING%BECKE_CONSTRAINT_A.

Keyword descriptions

STRENGTH
Force constants of the constraints. Give one value per constraint group.
This keyword expects a list of reals. Default value: 0.0000000000000000E+00
TARGET
TARGET {real}
Target values of the constraints. Give one value per constraint group.
This keyword expects a list of reals. Default value: 0.0000000000000000E+00
ADJUST_SIZE
ADJUST_SIZE
Adjust cell boundaries with covalent atomic radii to generate a heteronuclear cutoff profile. These radii are defined in the keyword ATOMIC_RADII.
This keyword expects a logical
This keyword behaves as a switch T . Default value: F
ATOMIC_RADII
ATOMIC_RADII {real} {real} {real}
Defines atomic radii to generate a heteronuclear cutoff profile. Give one value per element in the same order as they appear in the input coordinates.
This keyword expects a list of reals. Default unit: [angstrom].
SHOULD_SKIP
SHOULD_SKIP
If grid point is farther than GLOBAL_CUTOFF from all constraint atoms, move directly to next grid point, thus saving computational resources.
This keyword expects a logical
This keyword behaves as a switch T . Default value: F
ATOMIC_CHARGES
ATOMIC_CHARGES
Calculate atomic Becke charges. Note: If the number of atoms is greater than the default pw_pool max cache, calculation of atomic charges will prompt a warning during deallocation of atomic grids.
This keyword expects a logical
This keyword behaves as a switch T . Default value: F
CAVITY_CONFINE
CAVITY_CONFINE
Activates Gaussian cavity confinement. The constraint is evaluated only inside the cavity. The cavity is formed by summing spherical Gaussians centered on the constraint atoms.
This keyword expects a logical
This keyword behaves as a switch T . Default value: F
CAVITY_SHAPE
CAVITY_SHAPE (SINGLE|VDW|COVALENT|USER)
Specifies the type of Gaussian cavity used.
This keyword expects a keyword. Default value: DEFAULT
valid keywords:
  • DEFAULT: Use covalent radii (in angstrom) to construct Gaussians, but fixed 1.0_dp radius for elements with a radius larger than this value.
  • SINGLE: Single Gaussian for all atom types with radius given by CAVITY_RADIUS.
  • VDW: Use van der Waals radii to construct Gaussians.
  • COVALENT: Use covalent radii to construct Gaussians.
  • USER: Use user defined radii (keyword ATOMIC_RADII) to construct Gaussians.
CAVITY_USE_BOHR
CAVITY_USE_BOHR (SINGLE|VDW|COVALENT|USER)
Convert the cavity radius from angstrom to bohr. This results in a larger confinement cavity than without unit conversion.
This keyword expects a logical
This keyword behaves as a switch T . Default value: F
CAVITY_PRINT
CAVITY_PRINT
Print cavity in Gaussian cube file format. Currently, printing options are hardcoded.
This keyword expects a logical
This keyword behaves as a switch T . Default value: F
CAVITY_RADIUS
CAVITY_RADIUS
Parameter controlling the creation of Gaussian cavity confinement.
This keyword expects a real. Default unit: [angstrom], default value: 3.0000000000000000E+00 angstrom
EPS_CAVITY
EPS_CAVITY {real}
Density threshold for cavity creation. Grid points where the Gaussian density falls below the threshold are ignored.
This keyword expects a real. Default value: 9.9999999999999995E-07
CUTOFF_TYPE
CUTOFF_TYPE (GLOBAL|ELEMENT)
Specifies the type of cutoff used.
This keyword expects a keyword. Default value: GLOBAL
valid keywords:
  • GLOBAL: Use a single value for all elements. Read from GLOBAL_CUTOFF.
  • ELEMENT: Use a different value for all elements. Values read from ELEMENT_CUTOFF.
GLOBAL_CUTOFF
GLOBAL_CUTOFF
Parameter used to select which atoms contribute to the weight function at each real space grid point.
This keyword expects a real. Default unit: [angstrom], default value: 3.1750632515000001E+00 angstrom
ELEMENT_CUTOFF
ELEMENT_CUTOFF {real} {real} {real}
Defines element specific cutoffs to decide which atoms contribute to the weight function at each real space grid point. Give one value per element in the same order as they appear in the coordinates.
This keyword expects a list of reals. Default unit: [angstrom].
IN_MEMORY
IN_MEMORY
Precompute gradients due to Becke constraint during initial formation of constraint and store them in memory. Useful in combination with confinement, memory intensive otherwise. Does nothing if forces are not calculated.
This keyword expects a logical
This keyword behaves as a switch T . Default value: F
FRAGMENT_DENSITIES
IN_MEMORY
Use fragment densities as the target value of the constraint. Takes as input the electron densities of two isolated fragments in the same geometry that they have in the full system. The isolated fragment densities are read from cube files defined in FRAGMENT_{A,B}_FILE. With this keyword active, the constraint enforces that the number of electrons for both fragments is the same as they would have in the superposition of the isolated fragments. Supports only static calculations. Weight must be defined for one fragment only!
This keyword expects a logical
This keyword behaves as a switch T . Default value: F
FRAGMENT_A_FILE_NAME
FRAGMENT_A_FILE_NAME
Name of the reference electron density cube file for fragment A. May include a path. The reference electron density needs to be outputted on the same grid as the full system (same cutoff and cell, output stride 1).
This keyword expects a string. Default value: fragment_a.cube
FRAGMENT_B_FILE_NAME
Name of the reference electron density cube file for fragment A. May include a path. The reference electron density needs to be outputted on the same grid as the full system (same cutoff and cell, output stride 1).
This keyword expects a string. Default value: fragment_b.cube
