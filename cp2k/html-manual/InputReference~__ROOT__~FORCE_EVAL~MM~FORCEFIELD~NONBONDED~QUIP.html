Section QUIP

Index of all sections. This section is located at __ROOT__%FORCE_EVAL%MM%FORCEFIELD%NONBONDED%QUIP.

This section specifies the input parameters for QUIP potential type. Mainly intended for things like GAP corrections to DFT to achieve correlated-wavefunction-like accuracy. Requires linking with quip library from http://www.libatoms.org . This section can be repeated.

This section cites following reference: [QUIP2014]

Keyword descriptions

ATOMS
ATOMS {KIND1} {KIND2}
Defines the atomic kinds involved in the QUIP potential. For more than 2 elements, &QUIP section must be repeated until each element has been mentioned at least once. Set IGNORE_MISSING_CRITICAL_PARAMS to T in enclosing &FORCEFIELD section to avoid having to list every pair of elements separately.
This keyword expects precisely 2 words
PARM_FILE_NAME
PARM_FILE_NAME {FILENAME}
Specifies the filename that contains the QUIP potential.
This keyword expects a string. Default value: quip_params.xml
variants: PARMFILE
INIT_ARGS
INIT_ARGS
Specifies the potential initialization arguments for the QUIP potential. If blank (default) first potential defined in QUIP parameter file will be used.
This keyword expects a list of words. Default value: " "
CALC_ARGS
CALC_ARGS
Specifies the potential calculation arguments for the QUIP potential.
This keyword expects a list of words. Default value: " "
